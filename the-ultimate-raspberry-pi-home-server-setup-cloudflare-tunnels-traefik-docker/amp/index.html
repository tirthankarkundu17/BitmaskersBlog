The Ultimate Raspberry Pi Home Server Setup: Cloudflare Tunnels + Traefik + Docker

Tirthankar Kundu - 01 Mar 2026

If you've ever tried to host a web application from your home network on a Raspberry Pi, you know the struggle. You have to mess with your router's port forwarding, configure Dynamic DNS (DDNS) because your ISP keeps changing your IP address, and desperately hope you haven’t accidentally exposed your entire home network to hackers on the internet.

What if I told you there's a better way? A way that requires zero open router ports, automatically handles free SSL certificates, and lets you effortlessly add new apps to your server in seconds?

In this post, I'll walk you through setting up a bulletproof, highly-extensible Raspberry Pi hosting environment. We'll be using Cloudflare Tunnels running on bare-metal and Traefik running in Docker to securely expose a sample full-stack app (my very own URL unshortener, Unshorten-It).

Let's dive in.

The Architecture: How It Works

Here is a bird's-eye view of what we are building:

  1. Cloudflared (Bare Metal): The Cloudflare daemon connects your Pi securely to the Cloudflare Edge network. No inbound ports open!
  2. Docker Network: We create a single, unified Docker network called proxy.
  3. Traefik (Docker): Traefik listens to requests passing through the Cloudflare tunnel and dynamically routes them to the correct Docker container.
  4. Your Apps (Docker): Any app you spin up on the proxy network with the right Docker labels gets securely routed to the internet automatically.
Prerequisites
  • A Raspberry Pi (or any Linux machine) with Docker and Docker Compose installed.
  • A free Cloudflare account.
  • A domain name with its DNS managed by Cloudflare.
Step 1: Install Cloudflared (Bare Metal)

Instead of running Cloudflare inside Docker, we are going to run it directly on the Raspberry Pi OS. This keeps the secure tunnel stable at the OS-level and ensures your server connects to Cloudflare as soon as it boots up, independently of the Docker daemon.

Run the following commands on your Pi to install it:

# Download the ARM64 package (Assuming Pi 4 or Pi 5 on 64-bit OS)
wget -q https://github.com/cloudflare/cloudflared/releases/latest/download/cloudflared-linux-arm64.deb

# Install the package
sudo dpkg -i cloudflared-linux-arm64.deb

# Authenticate with Cloudflare
cloudflared tunnel login

(This will provide a URL for you to log into Cloudflare and authorize your domain).

Next, create a tunnel:

cloudflared tunnel create my-pi-tunnel

Important: Note the Tunnel ID. You'll need it for the next steps!

Finally, route your domain to the tunnel:

cloudflared tunnel route dns my-pi-tunnel subdomain.yourdomain.com
Step 2: Configure the Cloudflare Ingress Rules

Now, we need to tell cloudflared what to do when traffic hits unshorten.yourdomain.com.

Create a configuration file at ~/.cloudflared/config.yml (or /etc/cloudflared/config.yml):

tunnel: <YOUR-TUNNEL-ID>
credentials-file: /root/.cloudflared/<YOUR-TUNNEL-ID>.json

ingress:
  # Route traffic for this app into Traefik's port 80
  - hostname: subdomain.yourdomain.com
    service: http://localhost:80
  
  # (Optional) Wildcard route to send ALL subdomains into Traefik automatically!
  # - hostname: *.yourdomain.com
  #   service: http://localhost:80
    
  # Fallback rule
  - service: http_status:404

To make sure your tunnel runs in the background even if you restart your Pi:

sudo cloudflared service install
sudo systemctl start cloudflared
Step 3: Create the Unified Docker Network

We need a dedicated network so that Traefik can communicate with all the apps you eventually want to host. Let's call it proxy.

docker network create proxy
Step 4: The Magic docker-compose.yml

This is where it all comes together. We will deploy Traefik alongside our app (Unshorten-It, which has a FastAPI backend and a React frontend). Traefik will listen on port 80, intercept the traffic dumped by Cloudflare, and route it to the frontend or backend based on the URL path.

Create a  docker-compose.yml file:

networks:
  proxy:
    external: true

services:
  # 1. Traefik Reverse Proxy
  traefik:
    image: traefik:v2.10
    container_name: traefik
    command:
      - "--providers.docker=true"
      - "--providers.docker.exposedbydefault=false"
      - "--entrypoints.web.address=:80"
    ports:
      - "80:80"  # Exposed to the host so bare-metal cloudflared can push traffic here
    networks:
      - proxy
    volumes:
      - /var/run/docker.sock:/var/run/docker.sock:ro
    labels:
      - "traefik.enable=true"
    restart: unless-stopped

  # 2. FastAPI Backend
  backend:
    image: your_docker_username/unshorten-it-backend:latest
    container_name: unshorten-it-backend
    restart: unless-stopped
    networks:
      - proxy
    labels:
      - "traefik.enable=true"
      # Route all /api paths to backend
      - "traefik.http.routers.unshorten-api.rule=Host(`subdomain1.yourdomain.com`) && PathPrefix(`/api`)"
      - "traefik.http.routers.unshorten-api.entrypoints=web"

  # 3. React Frontend
  frontend:
    image: your_docker_username/unshorten-it-frontend:latest
    container_name: unshorten-it-frontend
    restart: unless-stopped
    environment:
      - API_BASE_URL=https://unshorten.yourdomain.com
    networks:
      - proxy
    labels:
      - "traefik.enable=true"
      # Route the root paths to the frontend
      - "traefik.http.routers.unshorten-ui.rule=Host(`subdomain1.yourdomain.com`)"
      - "traefik.http.routers.unshorten-ui.entrypoints=web"

Start up your cluster:

docker-compose up -d
The Result: Why This Setup is Unbeatable

If you navigate to https://subdomain.yourdomain.com, you'll see the app live, wrapped in a beautiful, free SSL certificate provided by Cloudflare.

But why go through the effort of combining Traefik and Cloudflare?

  1. Ultimate Extensibility: The single proxy network means whenever you want to self-host something new (like Nextcloud, a Postgres database, or an uptime monitor), you don't even need to touch Cloudflare configurations. Just spin up the new container in the proxy network and add Traefik labels. Traefik will dynamically pick it up!
  2. Absolute Security: No open ports on your router. Your Raspberry Pi forms an outbound connection to Cloudflare. If anyone tries to DDoS your home IP, they can't reach it.
  3. Modularity: Keeping cloudflared managed on the OS level (via systemd) allows your Pi to regain tunnel connectivity reliably during sudden reboots, even before the Docker daemon spins up.

Welcome to the future of self-hosting at home! 🏠🥂
