Automate Everything: How to Self-Host n8n on Raspberry Pi with Docker

Jan 31, 2026 3 min read raspberrypiaiautomation

If you’re into automation, you’ve probably heard of n8n. It’s the "fair-code" alternative to Zapier or Make.com that allows you to build complex workflows with a node-based visual editor.

While the cloud version is great, self-hosting n8n on a Raspberry Pi unlocks a massive advantage: total control. You get privacy, no execution caps, and—crucially for home lab enthusiasts—the ability to interact directly with local services and files without exposing them to the internet.

In this guide, we’ll deploy n8n using Docker Compose. This is the cleanest method, making updates and backups trivial.

The Gear
  • Raspberry Pi 4 or 5 (4GB+ RAM recommended for smooth performance).
  • Raspberry Pi OS (64-bit). Note: n8n requires a 64-bit OS. If you are on an older 32-bit install, you will likely encounter compatibility issues.
  • Docker & Docker Compose installed.
Step 1: Install Docker

If you haven't set up Docker on your Pi yet, the convenience script is the fastest way to get started.

Open your terminal and run

# Update your packages
sudo apt update && sudo apt upgrade -y

# Download and run the official Docker install script
curl -fsSL https://get.docker.com -o get-docker.sh
sudo sh get-docker.sh

# Add your user to the Docker group (so you don't need 'sudo' for every command)
sudo usermod -aG docker $USER

# Log out and back in for changes to take effect
logout
Step 2: Create the Directory Structure

We need a place to store our Docker configuration and a persistent volume so you don't lose your workflows when you restart the container.

Bash

# Create a folder for the project
mkdir -p ~/n8n-docker

# Create a folder for local data persistence
mkdir -p ~/n8n-docker/data

# Navigate into the folder
cd ~/n8n-docker
Step 3: The Docker Compose File

This is the heart of the setup. Create a docker-compose.yml file:

vim docker-compose.yml

Paste the following configuration. This setup uses the standard n8n image and maps the data folder we just created.

version: "3"

services:
  n8n:
    image: n8nio/n8n:latest
    container_name: n8n
    restart: unless-stopped
    ports:
      - "5678:5678"
    environment:
      - N8N_HOST=n8n.local
      - N8N_PORT=5678
      - N8N_PROTOCOL=http
      - GENERIC_TIMEZONE=Asia/Kolkata # Change to your timezone
      - N8N_METRICS=true # Optional: good for monitoring
    volumes:
      - ./data:/home/node/.n8n
    # Optional: If you plan to use local files/python scripts, mount those directories too
    # - /home/pi/scripts:/home/node/scripts
Step 4: Launch the Stack

With the file in place, spin it up:

docker compose up -d

Docker will pull the image and start the container in "detached" mode (background). You can check if it's running with:

docker compose logs -f

(Press Ctrl+C to exit the logs).

Step 5: Access Your Automation Station

Give it about a minute to initialize. Then, open a browser on any computer on your network and go to:

http://<YOUR-PI-IP-ADDRESS>:5678

You should see the setup screen asking you to create an owner account.

Once you’re in, you have a fully functional automation platform running on your desk. Next time you just sign in with the credentials -

Why This Setup is Powerful

Beyond saving money on subscriptions, self-hosting n8n on a Pi is incredible for local-first automation.

  1. Local LLMs: If you run local AI models (like Ollama) on your network, n8n can hit those API endpoints with zero latency and no cost per token.
  2. Filesystem Access: You can read/write files directly to the Pi.
  3. Python Integration: You can execute Python scripts directly within n8n nodes, leveraging the Pi's local libraries.
